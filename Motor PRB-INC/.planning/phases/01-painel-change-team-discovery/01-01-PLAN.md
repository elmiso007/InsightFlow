<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| DBA → schema lwsa | DDL alterando schema compartilhado — requer privilégio elevado |
| Operador → tabela master | INSERTs do seed manipulam dados que orquestram queries Python |
| Motor (psycopg2) → motor_change_team_painel | TRUNCATE requer ACCESS EXCLUSIVE lock — pode conflitar com leitores Superset |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | lwsa.motor_change_team (lista master) | accept | D-01 LOCKED: gestão manual via SQL direto. Auditoria mínima via `adicionado_em` + `removido_em` + `observacao`. Para auditoria forte (pgaudit) — fora de escopo. |
| T-01-02 | Denial of Service | lwsa.motor_change_team_painel TRUNCATE | mitigate | Documentar no header do DDL que TRUNCATE pode aguardar leitores Superset; cadência 6h tolera espera. Sem CASCADE (tabela é folha). RESTART IDENTITY zera serial. |
| T-01-03 | Tampering | sql/seed_change_team.sql (88 INSERTs) | mitigate | Idempotência via `IF NOT EXISTS (SELECT 1 ...)` por linha (Postgres 9.2 compat). Header documenta que para REMOVER PRB usa-se UPDATE ativo=false, não DELETE. |
| T-01-04 | Information Disclosure | descricao_curta no chart Superset | accept | Audiência do Superset já é restrita; mesma política dos charts motor_* existentes. |
| T-01-05 | Elevation of Privilege | conta motor com TRUNCATE em lwsa.motor_change_team_painel | mitigate | Validar com DBA que conta motor (já owner de lwsa.motor_*) tem TRUNCATE. Fallback documentado em RESEARCH §Environment Availability: usar DELETE FROM ... se TRUNCATE bloqueado. |
</threat_model>
